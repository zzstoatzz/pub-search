pub search / labels

pub-search runs a labeler that flags accounts publishing machine-generated bulk content (registry/feed mirrors, catalogs, database exports) rather than human writing — so they can be told apart from real authors. This is a read-only view of what it has decided; search doesn't act on it yet. Labeler identity: @pub-search.waow.tech.

how a label is decided
It decides in two steps, on its own:
  1. title-pattern pre-filter — every account's post titles are scored for how templated they are: a repeated machine pattern like S##E## … or … presented by … (digits shown as #). A score from 0–1; ≥ 0.50 makes the account a candidate. The score never labels anything by itself — it only decides what's worth a closer look.
  2. content review — for each candidate, a model (running on co/core) reads sample post content and judges whether it's automated output or a person writing. Only if it confirms "machine" does the labeler emit the bulk-mirror label.
labeled confirmed machine-generated — carries the bulk-mirror label.
rejected looked templated, but the model judged it human writing — not labeled.
vetoed has reader curation (someone recommends/subscribes) — never labeled, whatever the score.
pending flagged; awaiting content review.
Every label is reversible (a signed negation), so a wrong call is one correction away.
